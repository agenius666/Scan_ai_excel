# ScanExcel

一个面向安卓优先、兼顾跨平台演进的 Flutter 文档扫描 + AI 核验应用。

## 当前能力

- 导入 `.xlsx`
- 支持 Android 外部分享/打开 `.xlsx` 后导入
- 分享导入时弹出规则确认对话框
- 导入前检查 Sheet、表头行和关键列名
- 默认规则与当前活动规则分离：共享导入时可只影响当前会话，不污染默认配置
- 按规则解析任务列表
- 多页拍照扫描
- 拍后裁边编辑
- Android 侧接入 OpenCV 文档角点检测（需真机编译验证）
- 手动四角微调
- 四角透视矫正输出
- 图片旋转（左转 / 右转 / 180°）
- 本地生成 PDF
- 查看 PDF、分享 PDF、任务中记录的 PDF 来源统一指向同一个真实文件
- 调用多模态模型做字段核验
- 支持基础多协议接口分流：
  - OpenAI-compatible `/chat/completions`
  - Anthropic 风格 `/messages`
- 将 AI 核验结果写回 Excel
- 导出新的 `.xlsx`
- 导出成功后优先展示用户可读路径；本地文件路径时尝试直接调用系统打开
- 历史任务记录（SQLite）
- 核验队列页面
- 配置信息持久化保存
- Android 文件夹选择器选择导出目录
- 基于持久化 tree URI / DocumentFile / ContentResolver 的导出写入

## 当前主要页面

### 首页
- 首次导入 xlsx 后，任务列表主按钮显示为 **扫描**
- 已扫描任务主按钮显示为 **重扫**
- 未扫描任务默认不显示 **查看**
- 导出 xlsx 后：若返回的是本地文件路径，则尝试直接调用系统打开；若通过 Android 文件夹选择器导出，则展示用户可读保存路径，并优先走原生 document URI 打开
- 首页摘要卡片明确区分“当前会话规则”与默认配置，降低共享导入后的理解成本
- 首页、历史页、队列页、详情页的空状态统一为卡片式提示
- 可进入核验队列、历史任务
- 当前文件区域支持 **清除当前文件**，无需重启 App 即可回到未导入状态并重新调整配置

### 扫描页
- 支持连续拍摄多页
- 支持删除、重拍单页
- 完成后进入裁边/扫描确认流程

### 裁边页
- 自动默认裁边框
- 支持拖动四角微调
- 支持左转 / 右转 / 180° / 重置裁边
- 当前不包含彩色增强、黑白扫描等增强模式

### 扫描确认页
- 展示统一风格的任务信息头卡与扫描图片缩略图
- 支持点击缩略图查看原图
- 支持开始核验、重新扫描、生成 PDF、查看 PDF
- 生成后的 PDF 会作为该任务唯一的真实 PDF 来源，后续查看与分享均基于同一文件
- 原图预览可通过左上角返回，也可直接点击图片返回

### 任务查看页
- 展示统一风格的任务信息头卡与扫描图片缩略图
- 支持进入扫描确认页
- 原图预览可通过左上角返回，也可直接点击图片返回

### 结果详情页
- 展示字段比对
- 展示模型原始返回（默认折叠，可展开查看）
- 标题与状态文案统一为“核验结果”语义
- 底部仅保留返回按钮

## 存储策略

### 应用私有目录
用于内部数据：
- SQLite 历史记录数据库
- 应用配置
- 扫描过程中的内部文件
- 分享导入时的临时缓存文件

### 用户导出目录
用于用户可见导出文件：
- `xxx_MMddHHmmss.pdf`
- `xxx_已核验_MMddHHmmss.xlsx`

默认基础目录为 Download。用户也可以在“我的”页面通过 Android 文件夹选择器选择其他目录。
程序会在所选目录下自动创建 `ScanExcel/` 子文件夹，并通过持久化 tree URI 在该目录内写入文件。

## 分享导入当前行为

1. 用户从微信、QQ、文件管理器等外部 App 打开或分享 `.xlsx` 到 ScanExcel
2. App 启动后弹出“共享 Excel 导入确认”对话框
3. 用户确认或修改：
   - Sheet 名
   - 表头所在行
   - 任务名称列
   - PDF 命名列
   - 核验字段
   - 结果列名
4. 系统先做导入前检查：
   - Sheet 是否存在
   - 表头行是否有效
   - 任务名称列 / PDF 命名列 / 核验字段是否存在
   - 结果列名允许暂不存在，后续写回时会自动补列
5. 检查通过后再正式导入
6. 如需，用户可勾选“同时保存为默认规则”；不勾选时仅更新当前这次导入后会话内生效的规则，不覆盖默认规则；勾选后则同时保存为默认规则

## Android 运行前建议确认

- 已声明网络权限 `android.permission.INTERNET`
- 已声明相机权限 `android.permission.CAMERA`
- `minSdk` 满足依赖要求
- OpenCV Maven 依赖可正常解析
- 设备存在可打开 xlsx 的外部应用时，“导出后自动打开”体验更完整

## iOS 运行前建议确认

- `Info.plist` 中已配置 `NSCameraUsageDescription`
- 如继续推进 iOS OpenCV，需要补充对应原生依赖接入

## 使用建议

1. 导入 Excel，或从外部 App 共享/打开 xlsx 到 ScanExcel
2. 如为分享导入，先确认 Excel 规则
3. 在任务列表中逐条扫描纸质文件
4. 在裁边页完成旋转、裁边微调
5. 在扫描确认页开始核验，系统放入队列异步处理
6. 按需生成 PDF
7. 按需导出 `.xlsx`
8. 在任务查看、结果详情、历史任务中查看记录

## 当前主要防呆

- 设置页保存时会拦截以下明显错误：
  - 接口地址为空
  - 模型名为空
  - 超时秒数不是有效正整数
  - Sheet 名为空
  - 表头所在行不是大于 0 的整数
  - 任务名称列为空
  - PDF 命名列为空
  - 核验字段为空
  - 结果列名为空
- 若当前已经导入 Excel，保存设置时会拿当前工作簿做规则校验
- 结果列名允许在原始 Excel 中不存在；写回时自动补列
- 共享导入时可选择“只影响当前会话规则”或“同时保存为默认规则”
