# ScanExcel

一个面向安卓优先、兼顾跨平台演进的 Flutter 文档扫描 + AI 核验应用。

## 当前能力

- 导入 `.xlsx`
- 按规则解析任务列表
- 多页拍照扫描
- 拍后裁边编辑
- Android 侧接入 OpenCV 文档角点检测（需真机编译验证）
- 手动四角微调
- 四角透视矫正输出
- 图片旋转（左转 / 右转 / 180°）
- 本地生成 PDF
- 调用多模态模型做字段核验
- 支持基础多协议接口分流：
  - OpenAI-compatible `/chat/completions`
  - Anthropic 风格 `/messages`
- 将 AI 核验结果写回 Excel
- 导出新的 `.xlsx`
- 历史任务记录（SQLite）
- 核验队列页面
- 配置信息持久化保存

## 当前主要页面

### 首页
- 首次导入 xlsx 后，任务列表主按钮显示为 **扫描**
- 已扫描任务主按钮显示为 **重扫**
- 未扫描任务默认不显示 **查看**
- 可进入核验队列、历史任务

### 扫描页
- 支持连续拍摄多页
- 支持删除、重拍单页
- 完成后进入裁边/扫描确认流程

### 裁边页
- 自动默认裁边框
- 支持拖动四角微调
- 支持左转 / 右转 / 180° / 重置裁边
- 当前不包含彩色增强、黑白扫描等增强模式

### 扫描确认页
- 展示任务信息与扫描图片缩略图
- 支持点击缩略图查看原图
- 支持开始核验、重新扫描、生成 PDF、查看 PDF
- 原图预览可通过左上角返回，也可直接点击图片返回

### 任务查看页
- 展示任务信息与扫描图片缩略图
- 支持进入扫描确认页
- 原图预览可通过左上角返回，也可直接点击图片返回

### 结果详情页
- 展示字段比对
- 展示模型原始返回
- 底部仅保留返回按钮

## 存储策略

### 应用私有目录
用于内部数据：
- SQLite 历史记录数据库
- 应用配置
- 扫描过程中的内部文件

### Download 目录
用于用户可见导出文件：
- `xxx_时间戳.pdf`
- `xxx_已核验_时间戳.xlsx`

说明：当前直接导出到 Download 根目录，不再创建子文件夹。

## 扫描能力现状

### Android
- 已接入原生 OpenCV 轮廓检测尝试
- 检测失败时回退默认裁边框
- 支持四角拖拽微调
- 支持旋转后尽量保留当前裁边区域
- 保存时输出透视矫正结果

### iOS
- 已有原生桥接骨架
- 当前尚未接入真实 OpenCV 检测
- 目前使用默认裁边框 + 手动四角微调

- 多页图片拍摄

## Android 运行前建议确认

- 已声明网络权限 `android.permission.INTERNET`
- 已声明相机权限 `android.permission.CAMERA`
- `minSdk` 满足依赖要求
- OpenCV Maven 依赖可正常解析

## iOS 运行前建议确认

- `Info.plist` 中已配置 `NSCameraUsageDescription`
- 如继续推进 iOS OpenCV，需要补充对应原生依赖接入

## 使用建议

1. 导入 Excel
2. 在任务列表中逐条扫描纸质文件
3. 在裁边页完成旋转、裁边微调
4. 在扫描确认页开始核验，系统放入队列异步处理
5. 按需生成 PDF
6. 按需导出 `.xlsx`
7. 在任务查看、结果详情、历史任务中查看记录
